# Table.sort()

The `table.sort()` method sorts the elements of a table in-place and returns the sorted table. By default, it sorts elements in ascending order (alphabetically or numerically). You can provide a custom comparator function for more control.

## Syntax
```lua  
table.sort(tableData, comparator)
```

## Parameters

<Properties>
  <Property name="tableData" type="table">
    The table to sort.
  </Property>
  <Property name="comparator" type="function" optional="true">
    A function defining the sort order. It should accept two arguments and return `true` if the first argument should appear before the second.
  </Property>
</Properties>

## Description
- If no `comparator` is provided, the method defaults to sorting in ascending order (alphabetical for strings, numerical for numbers).
- The sorting is performed in-place, meaning the original table is modified.
- Sorting is not guaranteed for tables with non-numerical keys.

:::note
If the table contains mixed data types (e.g., strings and numbers), sorting behavior may be undefined.
:::

## Examples

### Sorting a Table of Numbers
This example demonstrates how to sort a table of numbers in ascending order.

<CodeGroup title="Example 1" label='Sorting Numbers in Ascending Order'>
```lua
local numbers = { 5, 2, 8, 1, 3 }
table.sort(numbers)

-- Output: numbers = { 1, 2, 3, 5, 8 }
```
</CodeGroup>

### Sorting a Table of Strings
When no comparator is provided, strings are sorted alphabetically.

<CodeGroup title="Example 2" label='Sorting Strings Alphabetically'>
```lua
local fruits = { "Banana", "Apple", "Cherry" }
table.sort(fruits)

-- Output: fruits = { "Apple", "Banana", "Cherry" }
```
</CodeGroup>

### Custom Sorting with a Comparator
This example shows how to use a custom comparator function to sort numbers in descending order.

<CodeGroup title="Example 3" label='Sorting Numbers in Descending Order'>
```lua
local numbers = { 5, 2, 8, 1, 3 }
table.sort(numbers, function(a, b)
    return a > b
end)

-- Output: numbers = { 8, 5, 3, 2, 1 }
```
</CodeGroup>

### Sorting a Table of Mixed Data Types
Sorting a table with mixed types may produce undefined behavior and should generally be avoided.

<CodeGroup title="Example 4" label='Sorting a Mixed Table'>
```lua
local mixed = { "Apple", 3, "Banana", 1 }
table.sort(mixed)

-- Output: Behavior is undefined.
```
</CodeGroup>
